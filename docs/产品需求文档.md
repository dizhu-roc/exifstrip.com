## 一、项目概述

本项目为一款 **PC 端 Web 应用**，用于在浏览器内**查看、编辑与删除**图片的 EXIF/元数据。所有处理在**前端完成**，图片不上传至服务端。支持多格式上传与查看，编辑与删除的能力因格式不同而有所约束。

---

## 二、产品形态

### 2.1 网站类型

- **平台**：PC 网站
- **交互**：支持键鼠与触屏，主要操作不依赖 hover，触屏可正常完成核心流程

### 2.2 技术栈与部署

- **前端框架**：Next.js
- **部署与进程管理**：PM2（用于启动、守护与重启 Next 应用）

### 2.3 核心能力

| 能力 | 说明 |
|------|------|
| **EXIF 查看** | 按维度/区块展示当前图片的 EXIF，含风险等级与区块色相（参见《EXIF 维度解析与字段说明》） |
| **EXIF 编辑** | 对当前图片的指定字段进行修改并导出为新文件；**仅支持 JPEG** |
| **EXIF 删除** | 按预设或规则 strip 元数据并导出；JPEG 支持多预设，其他格式仅支持「删除全部」 |

### 2.4 多语言（前期）

- **支持语言**：中文（zh-CN）、English（en）、日本語（ja）。
- **默认语言**：英文（en）。英文对应路径**无前缀**（如根路径即英文）。
- **首次打开 / 未手动选择过语言时**：根据**浏览器语言**自动选择展示语言；若浏览器语言在支持列表中则用对应语言，否则回退到英文。
- **用户手动选择语言后**：以**用户选择**为准，切换到所选语言；后续访问时优先使用用户上次选择（实现上可通过本地存储等方式持久化）。

### 2.5 整体布局与导航

- **版心宽度**：1200px；内容区居中，超出部分由视口决定。
- **查看 / 编辑 / 删除**：作为**页面内功能**呈现（如在上传区或 Metadata 区域内的操作入口），**不作为**主导航或 Tab。

页面自上而下结构如下：

| 顺序 | 模块 | 说明 |
|------|------|------|
| 1 | **导航栏** | 顶部导航，含品牌等；**语言切换器**位于导航栏**右侧**，形态为**下拉菜单 + Globe 图标**（详见 2.7）。 |
| 2 | **网站介绍** | 小卡片形式：我们是谁、你能做什么、什么是 EXIF、关于隐私、了解更多。 |
| 3 | **图片上传 + GPS 地图** | 上传区带**回显**与**图片列表**；同屏或相邻展示 **GPS 地图**（有定位数据时可用）。 |
| 4 | **Metadata 显示区域** | 按**维度横向排列**（与《EXIF 维度解析与字段说明》中区块对应）；**可横向滑动**；**固定高度**，高于该高度的区块**内部纵向滚动**。 |
| 5 | **四个卡片介绍** | 四个介绍用卡片，内容与顺序由产品/文案定。 |
| 6 | **Full EXIF reference table** | **可折叠**；展开后**固定高度**，表体**内部滚动**；以**表格**展示完整 EXIF 参考。 |
| 7 | **隐私介绍** | **可折叠**区块，用于隐私说明等。 |
| 8 | **底部栏** | 页脚，版权、链接等。 |

- 上述 2、5、6、7 为内容/说明类模块；3、4 为核心功能（上传、选图、Metadata 查看/编辑/删除）。编辑与删除入口在页面内（如 Metadata 区域或上传区附近）呈现，不单独占导航或 Tab。

### 2.6 上传区

- **交互**：支持**拖拽**和**点击**选择图片；多选时一次可选多张。
- **空状态文案**（需做多语言）：**「拖拽图片文件到这里，或点击选择图片(们)」**（或等价翻译：Drag image files here, or click to select image(s)）。
- **有图时的展示**：
  - **单张**：上传区**整体**作为该图的回显/预览区域。
  - **多张**：上传区切换为**图片列表**，**竖排**，每项为 **缩略图 + 文件名**；点击某项即选中为当前图，Metadata 区域与 GPS 地图等随当前图更新。

### 2.7 语言切换器

- **位置**：**导航栏右侧**。
- **形态**：**Globe 图标 + 下拉菜单**；下拉内为当前支持语言（中/英/日）选项，选择后切换语言并持久化（见 2.4）。

### 2.8 编辑模式

- **编辑模式**：**行内编辑**。即可编辑字段在 Metadata 显示区域内直接行内编辑，不单独弹出编辑面板或页面。

## 三、支持的图片格式与能力矩阵

**支持的图片格式**以 **exifreader**（本项目用于解析 EXIF 的库）官方支持为准，当前为：

| 格式 | 说明 |
|------|------|
| **JPEG** | 完整支持 Exif、IPTC、XMP、ICC、MPF、缩略图等 |
| **TIFF** | 支持 Exif、IPTC、XMP、ICC 等 |
| **PNG** | 支持 Exif、IPTC、XMP、ICC 及图像尺寸等 |
| **HEIC/HEIF** | 支持 Exif、XMP、ICC、缩略图等 |
| **AVIF** | 支持 Exif、XMP、ICC、缩略图等 |
| **WebP** | 支持 Exif、XMP、ICC、缩略图及图像尺寸等 |

上传时仅接受以上格式；解析与展示能力随格式不同（详见 exifreader 官方 Support table）。若 exifreader 后续版本增删格式，以库文档为准并同步更新本文档。

- **上传与查看**：支持上表所列全部格式（JPEG、TIFF、PNG、HEIC、AVIF、WebP）。
- **编辑**：**仅 JPEG**。非 JPEG 不提供编辑入口或明确提示「仅支持 JPEG」。
- **删除**：
  - **JPEG**：提供多种预设（如 full、privacy、gps-only、device-only），用户选择预设后对**当前单张**执行删除并导出。
  - **非 JPEG**：仅提供「删除全部」元数据一种方式，对**当前单张**执行；实现上可能通过重编码（如 Canvas 重绘）完成，需在文案中说明可能影响文件大小或编码。

## 四、上传与处理方式

| 维度 | 规则 |
|------|------|
| **上传** | 支持**批量选择**，一次可选多张图片上传到当前会话。 |
| **查看** | **单张**。即使用户上传多张，EXIF 面板只展示**当前选中那一张**的解析结果。 |
| **编辑** | **单张**。仅对当前选中的图片进行编辑并导出；仅当该图为 JPEG 时可用。 |
| **删除** | **单张**。仅对当前选中的图片应用所选预设（或「删除全部」）并导出；**不提供批量删除**。 |

即：批量仅用于**上传与切换图片**，查看、编辑、删除均针对「当前这一张」。

## 五、功能约束与规则（与现有文档对齐）

以下约束与《EXIF 维度解析与字段说明》保持一致，实现时以该文档为准。

### 5.1 查看

- 解析与展示的维度、区块、字段、风险等级与区块色相，按文档中「纯前端可解析的 EXIF 维度」「展示区块」「风险维度与色相」执行。
- 解析基于 exifreader 4.x（expanded: true, includeUnknown: true），未列入区块的 tag 可在 raw 中展示。

### 5.2 编辑

- **编辑模式**：**行内编辑**（见 2.8）；即可编辑字段在 Metadata 区域内直接编辑。
- 仅 JPEG；非 JPEG 不提供编辑或提示不支持。
- 可编辑/只读/仅展示的字段、ASCII/数字/有理数/下拉等规则，按文档中「可编辑与只读」「fieldId 与 piexif 映射」「valueToPiexif」「下拉选项」执行。
- 导出为单张文件（如 edited-YYYYMMDDHHmmss.jpg）。

### 5.3 删除

- **JPEG**：预设及删除范围按文档「清理预设」执行（如 full、privacy、gps-only、device-only）；始终保留 Orientation；按预设或按区块删除逻辑与文档一致。
- **非 JPEG**：仅提供「删除全部」；若通过重编码实现，需在 UI 或说明中提示可能改变文件大小或编码。

## 六、非功能需求（建议）

- **触屏**：列表/缩略图、选择当前图、查看/编辑/删除等主要操作均可通过点击/触控完成，避免仅 hover 才出现的控件。
- **性能**：大图或批量上传时，考虑解析与渲染性能（如懒解析、防抖）。
- **错误与提示**：非 JPEG 尝试编辑时给出明确提示；解析失败、导出失败时给出可理解的错误信息。
