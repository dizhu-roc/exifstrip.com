# 字体、字号与字重规范

> 基于 **Source Sans Pro**，适用于 EXIF 工具 PC 端界面。实现时可通过 CSS 变量或设计 token 落地。

---

## 一、字体族

### 1.1 主字体（无衬线）

```
Source Sans 3, "Source Sans Pro", -apple-system, BlinkMacSystemFont, "Segoe UI", Roboto, sans-serif
```

- **首选**：Source Sans 3（Source Sans Pro 的后续版本，可变字体、字重更全；若需兼容旧环境可用 Source Sans Pro）。
- **回退**：系统无衬线栈，保证离线或字体未加载时仍可读。

### 1.2 等宽字体（技术标识与 raw）

```
ui-monospace, "Cascadia Code", "SF Mono", "Consolas", "Liberation Mono", monospace
```

- **建议用等宽的**：ExifReader **tag 名**（如 Make、DateTimeOriginal、GPSLatitude）、**raw 列表中的键**、以及需要对齐的**纯数字/代码式**内容（如部分 raw value）。
- **不建议整段用等宽的**：用户可读的字段值（如 "Canon EOS R5"、日期时间、"1/500"、用户填的 Artist/UserComment）。这类用主字体（无衬线）更易读，仅在「强调这是 EXIF 键名」时对**键**使用等宽。

### 1.3 中文 / 日文（CJK）回退

- Source Sans Pro / Source Sans 3 无 CJK 字形，多语言（中/日）下会回退到系统默认（如苹方、微软雅黑、Noto Sans CJK）。
- 若希望中英日视觉统一，可在实现时为 `lang="zh"` / `lang="ja"` 设置备用字体，例如：`"Source Sans 3", "Noto Sans SC", "Noto Sans JP", sans-serif`（需按实际引入的字体调整）。当前规范以英文界面为基准，CJK 回退由实现决定。

## 二、字号阶梯

| 名称 | 字号 | rem | 建议用途 |
|------|------|-----|----------|
| **Display** | 28px | 1.75rem | 页面主标题、首屏大标题 |
| **H1** | 24px | 1.5rem | 区块标题（如「设备信息」「拍摄参数」） |
| **H2** | 20px | 1.25rem | 卡片标题、面板内分组标题 |
| **H3** | 18px | 1.125rem | 列表组标题、次要分组 |
| **Body** | 16px | 1rem | 正文、表单输入、表格内容、默认文案 |
| **Small** | 14px | 0.875rem | 辅助说明、标签、次要信息 |
| **Caption** | 12px | 0.75rem | 提示文字、图例、时间戳、占位符 |

- 根字体建议设为 **16px**（`html { font-size: 16px }`），其余用 `rem` 便于整体缩放与无障碍。
- 触屏场景下正文不建议小于 14px，以保证可读性。

---

## 三、字重

| 名称 | 数值 | 建议用途 |
|------|------|----------|
| **Light** | 300 | 大标题装饰、极少使用 |
| **Regular** | 400 | 正文、说明、默认状态 |
| **Medium** | 500 | 标签、表头、强调但不抢眼的文案 |
| **Semibold** | 600 | 小标题、按钮文字、导航当前项、重要标签 |
| **Bold** | 700 | 主标题、强强调、关键操作 |

- 默认正文使用 **Regular (400)**。
- 区块标题、导航、按钮优先用 **Semibold (600)**，需要更强层级时用 **Bold (700)**。
- **Light (300)** 仅在大字号、需要「轻」视觉时少量使用，避免长段落。

## 四、使用场景对照（推荐）

| 场景 | 字号 | 字重 | 备注 |
|------|------|------|------|
| 页面主标题 | Display (1.75rem) | Semibold / Bold | 如「EXIF 查看」「清理元数据」 |
| 右侧面板区块标题（device / location / dateTime 等） | H1 (1.5rem) | Semibold | 与《EXIF 维度解析与字段说明》中 BLOCK 对应 |
| 卡片标题、分组名 | H2 (1.25rem) | Semibold | 如预设名称、工具步骤标题 |
| 列表/表格组标题 | H3 (1.125rem) | Medium | 可选 |
| 字段标签（如 Make、Model） | Small (0.875rem) 或 Body (1rem) | Medium | 与字段值区分；EXIF 区块内见第五节 |
| 字段值、输入框内容 | Body (1rem) 或 Small | Regular | 默认；EXIF 区块内建议 Small |
| 原始值、tag 名、代码 | Body 或 Small | Regular | tag 名建议等宽，可读值用主字体 |
| 提示、说明、占位符 | Small (0.875rem) | Regular | 可适当降低对比度 |
| 时间戳、次要元信息 | Caption (0.75rem) | Regular | |
| 风险等级标签（high/medium/low） | Small | Semibold | 配合色相使用 |
| 主按钮文字 | Body | Semibold | |
| 次要按钮 / 链接 | Body 或 Small | Regular / Medium | |

## 五、EXIF 字段展示专用约定

针对《EXIF 维度解析与字段说明》中的区块与字段列表，建议如下，以保证右侧面板信息密度与可读性一致。

| 元素 | 字号 | 字重 | 字体 | 说明 |
|------|------|------|------|------|
| 区块标题（device / imageSettings / location 等） | H1 (1.5rem) | Semibold | 主字体 | 与 BLOCK_TITLE_KEYS 对应，可与区块风险边框配合 |
| 字段标签（Make、Model、拍摄时间 等） | Small (0.875rem) | Medium | 主字体 | 与字段值区分层级；若需与 raw 的 tag 名区分，可对「i18n 后的标签」用主字体、对「原始 tag 名」用等宽 Small |
| 字段值（只读展示） | Small (0.875rem) | Regular | 主字体 | 如 "Canon EOS R5"、日期时间、"1/500" 等可读内容用主字体 |
| 字段值（可编辑输入框 / 下拉） | Small 或 Body | Regular | 主字体 | 触屏下建议不小于 14px（Small），便于点击与填写 |
| raw 列表：tag 名（PascalCase） | Small (0.875rem) | Medium | **等宽** | 如 DateTimeOriginal、GPSLatitude，便于与 value 区分 |
| raw 列表：description / value | Small (0.875rem) | Regular | 主字体或等宽 | 短值用主字体；长串或明显「技术值」可用等宽，便于对齐 |
| 无数据占位（如 "—"） | Small | Regular | 主字体 | 可适当降低对比度 |
| 区块风险提示（与色相配合） | Small | Semibold | 主字体 | 与「风险维度与色相」一致 |

- **行高**：区块内「标签 + 值」行数多，建议行高约 **1.4**，在紧凑与可读之间折中。
- **等宽使用原则**：仅对「技术标识」（tag 名、raw 键）或需要纵向对齐的数值使用等宽；面向阅读的文案与值用主字体。

## 六、行高建议

| 字号档位 | 行高（倍数） | 说明 |
|----------|--------------|------|
| Display / H1 | 1.2 ~ 1.3 | 标题紧凑 |
| H2 / H3 | 1.3 ~ 1.4 | |
| Body | 1.5 | 正文可读性 |
| Small / Caption | 1.4 ~ 1.5 | 小字略松避免挤在一起 |
| EXIF 区块内（标签+值行） | 约 1.4 | 见第五节 |

## 七、与多语言的关系

- 中、英、日均以 **Source Sans Pro / Source Sans 3** 为主字体；若后续为中文单独选字体，可在此文档增加「中文主字体」一节，并约定优先级（如中文用 Noto Sans SC，英文/日文用 Source Sans Pro）。
- 字号与字重规范不区分语言，统一按上表执行，便于实现与视觉一致。

---

## 八、实现与维护

- 建议在项目中用 **CSS 变量** 或 **设计 token** 暴露：`--font-sans`、`--font-mono`、`--text-display`、`--text-h1` … `--text-caption`、`--font-weight-*` 等，组件与页面引用变量而非写死字号/字重。
- 若使用 Tailwind：可扩展 `theme.fontFamily`、`theme.fontSize`、`theme.fontWeight` 与上述阶梯对齐。
- 文档版本与产品需求文档保持一致；字体、字号或字重若有调整，在此文档修订并注明日期。
