## 一、已实现功能概览

### 1. 网站与路由

| 能力 | 说明 |
|------|------|
| 首页 | `/`，集成上传、元数据、地图、介绍卡片、EXIF 参考表、隐私说明、FAQ 摘要等 |
| 指南 | `/guides`、动态路由 `/guides/[id]`，结构化指南内容 |
| 博客 | `/blog`、动态路由 `/blog/[id]`，文章列表与详情 |
| 静态页 | `/about`、`/faq`、`/privacy`、`/terms` |
| SEO | `sitemap.xml`（静态页 + 全部 guides/blog 条目）、`robots.txt` 指向站点地图 |
| 根布局元数据 | `metadataBase`、Open Graph、Twitter、canonical、结构化数据（WebApplication JSON-LD） |

### 2. 首页核心工具（浏览器端处理）

| 能力 | 说明 |
|------|------|
| 上传 | 点击选择、拖拽；多图列表（单次最多 10 张、单文件约 20MB 上限，以自检文档为准） |
| 支持格式（查看/解析） | JPEG、PNG、WebP、TIFF、HEIC/HEIF、AVIF 等（以 exifreader 支持为准） |
| 文件信息 | 上传后展示类型、尺寸、大小等基础信息 |
| 元数据展示 | 按维度分块（Device、Location、DateTime、Image settings 等），横向滑动；固定高度内纵向滚动 |
| 多图 | 列表切换、Prev/Next；Metadata 与地图随当前选中图更新 |
| 导出 EXIF | CSV、JSON、TXT；JPEG 另支持 HEX 导出 |
| 剥离（Strip） | JPEG：预设 Full / Privacy / GPS only / Device only；非 JPEG：仅「删除全部」类能力 |
| 下载清理后文件 | 剥离后可下载，文件名含 `_cleaned_` 等约定 |
| 编辑与删除（JPEG） | 行内编辑可编辑字段、删除可删字段；支持「Download with edits」导出 `_edited_` 文件 |
| GPS 地图 | 有 GPS 时在地图上展示位置与标记；无数据时占位文案 |
| 重置 | 清空列表与状态 |
| 解析失败 | 列表项错误提示，避免整页崩溃 |
| 快捷导航 | 右侧锚点：上传、元数据、隐私说明等区块 |

### 3. 国际化

| 能力 | 说明 |
|------|------|
| 语言 | English、中文（zh-CN）、日本語（ja） |
| 切换 | 导航栏 Globe 下拉；选择后持久化（本地存储） |
| 默认语言策略 | 与 PRD 一致：浏览器语言匹配则选用，否则英文等 |

### 4. 导航与页脚

| 能力 | 说明 |
|------|------|
| 顶栏 | Home、Guides、Blog、FAQ、About + 语言切换 |
| 页脚 | 站点介绍、Privacy / Terms / About / FAQ；外链：X（Twitter）、GitHub（以当前 `Footer.tsx` 为准） |

### 5. 分析与部署配套（工程侧）

| 能力 | 说明 |
|------|------|
| Google Analytics | 根布局通过 `NEXT_PUBLIC_GA_ID` 条件加载 gtag（需在 **build** 前配置，见 `.env.production.example`） |
| PM2 | 提供 `ecosystem.config.cjs` 示例 |
| Nginx | 仓库内 `exifstrip.conf` 为反向代理与域名跳转参考（非 Next 源码一部分） |

### 6. 测试与质量

| 能力 | 说明 |
|------|------|
| Playwright | `npm run test:ui`：首页区块、子页跳转、语言切换、上传与导出/剥离等（详见《功能自检文档》「当前 E2E 覆盖」） |
| 自检清单 | 《功能自检文档》覆盖拖拽、多图、编辑、地图、异常 Toast 等，部分建议人工回归 |

## 三、未来功能规划（建议）

以下按**价值 / 成本**大致排序，实施时按排期裁剪。

### 3.1 产品体验（短期～中期）

1. **反馈渠道落地**  
   页脚或关于页：邮箱、表单或 GitHub Issues 链接，与推广话术「欢迎提意见」一致。

2. **批量操作**  
   多图场景下「对全部图片执行同一 Strip 预设并打包下载」或逐张队列提示，降低重复操作成本。

3. **编辑体验对齐方案**  
   按《EXIF编辑功能方案》将 Orientation、Flash、GPS 经纬度参考等改为下拉/专用控件与校验，减少无效 EXIF 写入。

4. **无障碍与键盘**  
   焦点顺序、aria-label、跳过链接；地图与上传区对读屏友好说明。

5. **错误与边界提示统一**  
   导出/剥离/下载失败 Toast 与文案在多语言下一致（自检文档 12.x）。

### 3.2 内容与增长（持续）

6. **指南/博客扩充**  
   围绕长尾关键词与真实场景（社交平台发图前、工作流备份 EXIF 等）持续加文，与现有 `guidesStructure` / `blogStructure` 一致维护。

7. **结构化数据扩展**  
   对重点指南页、文章页增加 `Article` / `HowTo` 等 JSON-LD（在合规、不夸大前提下）。

### 3.3 技术形态（中期～长期）

8. **E2E 覆盖补全**  
   拖拽、Prev/Next、JPEG 编辑/删除整条链路、地图、失败 Toast 等纳入自动化，减少回归成本。

9. **PWA / 离线说明页**  
   仅缓存壳与静态页，核心仍本地处理图片；需注意 Service Worker 与大包体积。

10. **性能与分包**  
    Leaflet、exif 相关库按路由或动态 import 再评估，首屏与 TTFB 监控（配合 GA 或 Web Vitals）。

11. **非 JPEG 的编辑能力**  
    PNG/WebP 等写入 XMP 或厂商段能力差异大，需单独调研与兼容性测试，优先级通常低于 JPEG 体验打磨。

### 3.4 可选探索（按需）

12. **浏览器扩展**  
    右键「用 ExifStrip 打开」或剪贴板图片，引流至站点；开发与上架成本较高。

13. **开源协作**  
    README 贡献指南、issue 模板，与 GitHub 仓库联动。
