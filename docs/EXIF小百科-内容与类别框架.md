# EXIF 小百科 · 内容与类别框架

> 用于规划站内「EXIF 小百科 / 指南」的内容结构，便于后续撰写文章或搭建博客/帮助栏目。内容需基于公开标准与一手资料自行撰写，勿直接抓取第三方站点。

---

## 一、定位与受众


| 项目        | 说明                                              |
| --------- | ----------------------------------------------- |
| **名称**    | EXIF 小百科（或：EXIF 指南 / EXIF 入门）                   |
| **目的**    | 帮助用户理解什么是 EXIF、为何要关注、如何在本站查看与清理；提升信任与留存，兼顾 SEO。 |
| **受众**    | 普通拍照用户（手机/相机）、对隐私或元数据好奇的用户、需要快速查字段的进阶用户。        |
| **与本站关系** | 与首页「网站介绍卡片」「EXIF 参考表」「FAQ」互补；可挂在导航「博客」或独立「指南」下。 |
| **语言**    | 与本站一致：中文（zh-CN）、English（en）、日本語（ja）。            |

### 1. 认识 EXIF


| 序号  | 条目 ID               | 标题建议         | 内容要点                                                                               |
| --- | ------------------- | ------------ | ---------------------------------------------------------------------------------- |
| 1.1 | `what-is-exif`      | 什么是 EXIF？    | 定义（可交换图像文件格式）、谁制定与维护（JEIDA/JEITA）、常见嵌入在哪些格式（JPEG、TIFF、HEIC 等）；EXIF 是元数据标准而非文件格式本身。 |
| 1.2 | `exif-history`      | EXIF 的由来与演变  | 简要历史（如 1995 年至今）、版本演进（2.1 / 2.2 / 2.3 等）与主要新增内容；说明「标准公开、各设备普遍支持」。                  |
| 1.3 | `supported-formats` | 哪些格式包含 EXIF？ | JPEG、PNG、TIFF、HEIC/HEIF、AVIF、WebP 等在本站的支持情况；哪些可读、哪些可编辑/剥离，与首页「支持格式」一致。             |

### 2. EXIF 里有什么（按维度）

> 维度命名与 `src/constants/exifBlocks.ts`、`docs/EXIF维度解析与字段说明.md` 保持一致，便于与本站 Metadata 展示和参考表联动。


| 序号  | 条目 ID                | 标题建议          | 内容要点                                                                        |
| --- | -------------------- | ------------- | --------------------------------------------------------------------------- |
| 2.1 | `exif-structure`     | EXIF 的结构是怎样的？ | 高层概念：IFD（图像文件目录）、0th/Exif/GPS 等子块、Tag 与取值；不要求二进制级细节，侧重「有哪些大类信息」。            |
| 2.2 | `dimension-device`   | 设备与相机信息       | Make、Model、机身/镜头序列号、Software、固件等；说明由设备自动写入、可用来识别机型。                         |
| 2.3 | `dimension-settings` | 拍摄参数          | 光圈、快门、ISO、焦距、白平衡、闪光灯、曝光程序等；面向非专业用户做通俗解释，可举例。                                |
| 2.4 | `dimension-location` | 位置信息（GPS）     | 经纬度、海拔、时间戳等；说明由具备定位功能的设备写入；与下文「隐私」呼应。                                       |
| 2.5 | `dimension-datetime` | 日期与时间         | 拍摄时间、修改时间、数字化时间等；格式与用途简述；可提一句「无时区」的常见限制。                                    |
| 2.6 | `dimension-image`    | 图像属性          | 宽高、分辨率、色彩空间、位深度等；与「图像如何显示、打印」相关。                                            |
| 2.7 | `dimension-other`    | 作者、版权与注释      | Artist、Copyright、UserComment、ImageDescription、MakerNote 等；说明部分为可编辑、部分为厂商私有。 |

### 3. EXIF 与隐私


| 序号  | 条目 ID              | 标题建议              | 内容要点                                                        |
| --- | ------------------ | ----------------- | ----------------------------------------------------------- |
| 3.1 | `why-care-privacy` | 为什么要重视 EXIF 里的隐私？ | 与首页卡片 4 一致：照片常带 GPS、设备型号等个人隐私信息；未加处理就分享可能被他人获取；分享前检查与清理有必要。 |
| 3.2 | `sensitive-fields` | 哪些 EXIF 信息比较敏感？   | GPS、设备/镜头序列号、拍摄时间、UserComment 等；分点说明风险场景（如位置暴露、设备被识别）。      |
| 3.3 | `before-you-share` | 分享前建议怎么做？         | 先查看再决定是否剥离；按需使用「仅移除位置」「仅移除设备」或「全部移除」等策略；可引导至本站工具。           |

### 4. 如何查看与编辑 EXIF


| 序号  | 条目 ID                 | 标题建议            | 内容要点                                                                |
| --- | --------------------- | --------------- | ------------------------------------------------------------------- |
| 4.1 | `how-to-view`         | 如何查看 EXIF？      | 常见方式：系统自带预览、专业软件、在线工具；强调本站：浏览器内查看、不上传、按维度展示。                        |
| 4.2 | `how-to-edit-strip`   | 如何编辑或删除 EXIF？   | 编辑：部分字段可改（如 JPEG 下）；删除/剥离：按预设批量删除、或整段清除；非 JPEG 通过重编码去除。与本站「剥离预设」对应。 |
| 4.3 | `what-exifstrip-does` | ExifStrip 能做什么？ | 汇总本站能力：查看、按维度展示、按预设剥离、导出 EXIF 文本/十六进制、GPS 地图等；本地处理、免费、隐私优先。         |

### 5. 常见问题（FAQ）


| 序号  | 条目 ID                | 标题建议                     | 内容要点                                        |
| --- | -------------------- | ------------------------ | ------------------------------------------- |
| 5.1 | `faq-remove-safe`    | 删除 EXIF 会影响画质吗？          | 仅删元数据不重编码则不影响；非 JPEG 重编码时可能略有影响，可简述。        |
| 5.2 | `faq-who-writes`     | EXIF 是谁写进去的？             | 相机/手机拍摄时自动写入；部分软件编辑后会修改或追加。                 |
| 5.3 | `faq-social-network` | 发到社交平台后 EXIF 还在吗？        | 多数平台会剥离或改写，但不保证；建议先自行清理再分享。                 |
| 5.4 | `faq-can-recover`    | 删除后能恢复吗？                 | 一旦删除且未备份，无法从该文件中恢复；提醒备份原图。                  |
| 5.5 | `faq-raw-heic`       | RAW / HEIC 的 EXIF 有什么特别？ | 简述格式差异、本站对 HEIC 等支持情况；RAW 可仅作简要说明或链至「支持格式」。 |


> 与现有 `FAQSection`、`/faq` 页面可做交叉引用或合并，避免重复；小百科内 FAQ 可更偏「概念」，站内 FAQ 偏「使用与策略」。

## 五、撰写与维护约定

1. **原创表达**：基于公开标准（JEITA、CIPA、Wikipedia 等）自行组织语言，不整段摘抄第三方站点。
2. **事实一致**：技术事实（Tag 名、IFD、格式支持）与 `exifBlocks`、`stripExifPresets`、产品行为一致。
3. **口径一致**：隐私、本地处理、免费等表述与首页与隐私页一致。
4. **修订记录**：对框架或条目有增删改时，可在本文末尾更新「修订记录」表。
