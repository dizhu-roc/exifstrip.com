# 博客 · 内容与类别框架

> 用于规划站内「博客」栏目的内容结构与选题方向。博客侧重使用场景、技巧、产品动态与延伸话题，与「EXIF 小百科」形成互补：小百科负责系统性的概念与参考，博客负责灵活、时效性的内容。

---

## 一、定位与受众

| 项目 | 说明 |
|------|------|
| **名称** | 博客 / Blog |
| **目的** | 帮助用户用好 ExifStrip、了解产品与相关动态；传递使用技巧、场景与延伸话题；兼顾 SEO 与留存。 |
| **受众** | 使用或关注 ExifStrip 的用户；对照片隐私、EXIF 清理有需求的一般用户。 |
| **与小百科区别** | 小百科：系统性知识（什么是、结构、字段、隐私原理）；博客：怎么用、有哪些场景、最近有什么新东西。 |
| **与本站关系** | 首页第五张卡片「了解更多」链接至博客；导航栏「博客」入口。 |
| **语言** | 与本站一致：中文（zh-CN）、English（en）、日本語（ja）。 |

---

## 二、一级分类与二级选题

以下为建议的博客分类及可选文章选题。每项附「内容方向」，便于后续按选题撰写。选题可按需增删，不强求全部产出。

---

### 1. 使用技巧

| 序号 | 选题 ID | 标题建议 | 内容方向 |
|------|---------|----------|----------|
| 1.1 | `tip-before-travel-share` | 出行照片分享前，先清理 EXIF 再发 | 场景：旅行、出差拍的照片常有 GPS；分享到社交前用 ExifStrip 做「仅移除 GPS」或「仅隐私相关」；步骤简述 + 工具引导。 |
| 1.2 | `tip-batch-clean` | 批量处理多张照片的 EXIF：怎么更快？ | 场景：一次分享多张图；上传多张、按预设批量剥离、逐一下载；提示「继续添加」、预设选择。 |
| 1.3 | `tip-keep-what-you-need` | 只想删 GPS，其他信息保留：怎么做？ | 说明「仅移除 GPS」预设；适用场景：想保留设备、拍摄参数，但不想暴露位置。 |
| 1.4 | `tip-export-exif` | 先查看再决定：导出 EXIF 做备份或审计 | 场景：想备份元数据、或分享前先审计；说明导出为文本/十六进制的用法。 |

## 三、与百科、FAQ 的重复与处理

以下选题与百科或 FAQ 在主题上有重叠，建议**角度区分**或**合并/链接**，避免同一结论写两遍。

| 博客选题 | 百科/FAQ 对应 | 重叠点 | 建议 |
|----------|----------------|--------|------|
| 2.1 `scene-social-media` 发到朋友圈/微博后 EXIF 还在吗 | 百科 FAQ 5.3「发到社交平台后 EXIF 还在吗」 | 结论一致：多数平台会剥离，但不保证 | 博客做**场景化**：分平台简述处理差异、举例「先清再发」流程；文末链至 FAQ。百科/FAQ 保留简短结论即可。 |
| 4.2 `topic-platform-strip` 社交平台会怎么处理 EXIF | 同上 + 百科 3.3「分享前建议怎么做」 | 平台策略 + 建议先自行清理 | 博客可深入**各平台策略**（引用公开信息）；百科保留「先查看再决定、先清再发」策略。两处互相链接。 |
| 3.2 `product-supported-formats` ExifStrip 支持格式说明 | 百科 1.3「哪些格式包含 EXIF」 | 格式列表、本站支持能力 | **百科**：概念 + 哪些格式能带 EXIF + 本站读/写/剥离矩阵。**博客**：产品向、随版本更新的「格式支持说明」或合并进 `product-update`，避免单独一篇与百科同质。 |
| 1.3 `tip-keep-what-you-need` 只想删 GPS 其他保留 | 百科 4.2「如何编辑或删除」、3.3「分享前建议」 | 「仅移除 GPS」预设的用法 | 百科已讲预设与策略；博客做**场景+步骤+截图**，不重复概念，文末链至百科 4.2 / 3.3。 |
| 4.3 `topic-exif-vs-metadata` EXIF 和元数据是一回事吗 | 百科 1.1「什么是 EXIF」（EXIF 是元数据标准） | 概念边界 | 百科已定义；博客做**延伸**：EXIF vs IPTC vs XMP、常见软件写入哪些，链至百科 1.1。 |

**原则**：同一结论或同一概念只在百科/FAQ 写透一处；博客做场景、案例、平台差异或产品动态的延伸，并链回百科/FAQ。

在现有四类基础上，可酌情增加以下选题，用于丰富场景与时效内容，且不与百科系统知识重复。

| 分类 | 选题 ID（建议） | 标题建议 | 内容方向 |
|------|-----------------|----------|----------|
| **使用技巧** | `tip-archive-exif` | 照片归档前，用 EXIF 做一次整理与备份 | 场景：个人/家庭照片库；导出 EXIF 备份、按日期/设备筛选；与本站「导出」功能衔接。 |
| **使用技巧** | `tip-compare-before-after` | 清理前后对比：导出 EXIF 做审计 | 清理前导出一份、清理后再导出或对比；适合需要留痕或合规审计的场景。 |
| **场景与故事** | `scene-marketplace-photo` | 网店/二手交易：上传商品图前先剥 EXIF | 场景：闲鱼、淘宝、二手平台；他人转手图可能带原主设备/位置；建议上传前用本站查看与剥离。 |
| **场景与故事** | `scene-kids-photo-share` | 儿童照片分享前的 EXIF 习惯 | 场景：家长在班级群、社交平台发孩子照片；位置、时间等敏感；简短建议：先查看、按需剥离，链至百科隐私与分享前建议。 |
| **场景与故事** | `scene-screenshot-exif` | 截图与会议画面里也有 EXIF 吗？ | 澄清：截图多数不含 GPS，但可能含设备/时间；屏幕截图、会议软件保存的图可简要说明，引导「不确定就先查一下」。 |
| **产品与更新** | `product-roadmap` 或合并进 update | 计划中的功能 / 与同类工具的差异 | 可选：路线图简述、或「为什么我们强调本地、不上传」与同类在线工具的对比；随产品阶段更新。 |
| **延伸话题** | `topic-camera-disable-location` | 相机/手机里如何关闭或限制「位置写入照片」 | 系统设置层面：关闭照片定位、仅用 WiFi 定位等；与 EXIF 互补，不写技术细节，链至百科「位置信息」。 |
| **延伸话题** | `topic-copyright-in-exif` | 版权与作者信息：EXIF 里的著作权字段 | Artist、Copyright 等字段的用途；摄影师/机构署名与维权；链至百科「作者、版权与注释」维度。 |

以上为**可选补充**，不要求全部产出；优先写与本站工具强相关、且百科未展开的场景与技巧。

## 五、与小百科、FAQ 的关系（总表）

| 内容类型 | 归属 | 说明 |
|----------|------|------|
| 概念、结构、字段、标准 | 小百科 | 系统性、参考型 |
| 使用场景、技巧、产品动态、延伸话题 | 博客 | 灵活、时效型 |
| 高频问答（删除安全吗、能恢复吗等） | FAQ | 简短、可直接在站内展示 |

博客文章可在文末链至小百科相关条目或 FAQ；小百科可在「延伸阅读」链至博客。避免同一内容在三处重复撰写，优先在一处写透，他处做简要概括 + 链接。

## 六、内容形式与路由建议

| 项目 | 建议 |
|------|------|
| **形式** | 每篇独立文章；可配图（截图、示意图）；约 500～1500 字，视选题而定。 |
| **路由** | `/blog` 列表页；`/blog/[slug]` 文章页；slug 可与选题 ID 一致（如 `/blog/tip-before-travel-share`）。 |
| **发布时间** | 按实际撰写与发布填写；无固定频率要求。 |
| **多语言** | 与本站 locale 一致；每语种独立文章或共用 slug + 按 locale 切换内容。 |
