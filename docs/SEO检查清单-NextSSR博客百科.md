# SEO 检查清单与步骤（Next.js SSR｜博客/百科为主）

适用场景：英文海外市场；站点以**博客**与**百科/词典式**页面为主要自然搜索流量载体；技术栈为 **Next.js 且页面为 SSR**；已接入 **Google Search Console（GSC）** 与 **GA4**。

使用方式：按「步骤总览」顺序执行；每个维度下按检查项勾选。**检查规则**列给出可判定的通过/不通过与处理方向（不是保证排名，而是降低技术性自我伤害并提高可理解与可点击性）。

**完整度说明（边界）**：本文以**站内可审计**为主（技术 + 内容 + 衡量）；不包含外链拓展、付费投放、邮件/社交增长等站外策略。下列维度为“常见缺口补丁”；若业务不涉及（如无标签归档），对应项标为 N/A 即可。

---

## 步骤总览（建议顺序）

1. **抽样选取 URL**：从 GSC「成效」导出 Top 50 着陆页；再各抽 5 条：最新发布、最旧仍带来流量、随机百科页、随机博客分类页（如有）、**标签/作者归档页（如有）**、带查询参数或分页的 URL（如有）。  
2. **建立对照表**：每条 URL 记录：索引状态、canonical、title/h1、主要关键词意图（信息型/工具型/混合型）、GA4 关键事件（滚动/站内搜索/转化如有）。  
3. **先跑“阻断项”**：不可索引、canonical 错误、大面积重复/软 404、SSR 输出与浏览器严重不一致。  
4. **再跑“体验与理解项”**：CWV、结构化数据、内链、内容意图与 E‑E‑A‑T、**内容自然度（避免明显“写给 Google”）**。  
5. **闭环**：把 GSC（曝光/点击/排名）与 GA4（参与率/转化）按着陆页 Join，形成改版优先级队列。

## 维度 1：索引与抓取（GSC：编制索引 / 网页）

### 检查项

- **1.1 重要着陆页索引状态**  
  - **检查规则（通过）**：博客文章页、百科词条页在 GSC URL 检查中为「已编入索引」或等价成功状态；非刻意屏蔽页不应长期停留在「已发现 - 尚未编入索引」「已抓取 - 尚未编入索引」且无合理原因。  
  - **检查规则（不通过）**：核心流量页出现「被“noindex”标记排除」「因重定向被排除」「重复网页（用户未选定规范网页）」「软 404」等；需在模板层或内容层修正。

- **1.2 Sitemap 覆盖与噪声**  
  - **检查规则（通过）**：`sitemap.xml` 覆盖主要可索引的博客/百科 URL；不包含明显不应索引的页面（如无意义筛选结果、空状态页、测试页）。  
  - **检查规则（不通过）**：sitemap 大量 404/301、或包含与 `robots`/页面级指令冲突的 URL。

- **1.3 robots.txt 与页面级指令一致**  
  - **检查规则（通过）**：`robots.txt` 不误伤内容路径；页面级 `noindex` 仅用于确实不应进入索引的页面（如感谢页、站内搜索结果、重复打印版等）。  
  - **检查规则（不通过）**：内容页被 Disallow 或 `noindex` 与业务目标冲突。

- **1.4 抓取异常与状态码**  
  - **检查规则（通过）**：GSC「网页」中服务器错误（5xx）与重要 404 对核心模板不构成持续趋势。  
  - **检查规则（不通过）**：模板或数据源故障导致同类 URL 批量不可访问或返回错误内容。

- **1.5 手动操作、安全问题与移除请求（阻断项）**  
  - **检查规则（通过）**：GSC「手动操作」「安全问题」无未处理项；**移除**工具无长期误伤生产内容 URL 的遗留请求。  
  - **检查规则（不通过）**：存在未处理处罚/被黑注入/恶意重定向等；或误提交永久移除导致流量断崖。

### 操作建议（工具）

- GSC：**编制索引**、**网页**、**Sitemaps**、**robots.txt 测试**（如有）、单 URL **网址检查**。  
- 站点侧：确认 Next `generateMetadata`/head 输出与路由一致；确认动态路由 slug 变更策略（改名是否 301）。

## 维度 2：规范化与重复内容（canonical / 参数 / 分页）

### 检查项

- **2.1 Canonical 指向**  
  - **检查规则（通过）**：每个可索引页面 `<link rel="canonical">` 指向**自身首选 URL**（scheme/host/path 一致）；跨语言或跨版本不在此维度混用错误目标。  
  - **检查规则（不通过）**：canonical 指向首页/分类页“吸走”文章页权重；或指向 HTTP/错误主机名；或链到 404。

- **2.2 参数化 URL 与追踪参数**  
  - **检查规则（通过）**：`utm_*`、`gclid` 等追踪参数不产生大量“可索引重复”；canonical 仍指向干净 URL（或参数页明确 noindex）。  
  - **检查规则（不通过）**：同一文章存在多个被索引的带参版本且 canonical 未统一。

- **2.3 分页与系列页**  
  - **检查规则（通过）**：分页使用明确关系（`rel="next/prev"` 非必须，但需有清晰 crawl 路径）；canonical 策略与站点信息架构文档一致（通常每页自指或按你定义的规范）。  
  - **检查规则（不通过）**：分页页与第 1 页内容高度重复且都被索引、无规范化策略。

- **2.4 打印/分享/预览路由**  
  - **检查规则（通过）**：衍生路由要么 `noindex`，要么 canonical 回主文。  
  - **检查规则（不通过）**：衍生路由与主文同时竞争索引。

- **2.5 主机名、协议与路径形态统一（www / HTTPS / 尾斜杠）**  
  - **检查规则（通过）**：全站对外只推广**一种**规范形态：`https`、固定 `www` 或裸域二选一、**尾斜杠策略**（有或无）与 Next 路由配置一致；非规范访问统一 **301** 到规范 URL，且与 canonical 目标一致。  
  - **检查规则（不通过）**：同一内容可在多个主机/斜杠变体下 200 访问且无统一规范；或 301 链过长、环重定向。

- **2.6 重定向链与“假首页”**  
  - **检查规则（通过）**：重要 URL 重定向链尽量 ≤1–2 跳；避免用 **meta refresh** 或 JS 作为主要 canonical 手段替代 301。  
  - **检查规则（不通过）**：长链、临时 302 滥用、或内容页被误重定向到无关聚合页。

## 维度 3：Next.js SSR 与 HTML 可理解性（爬虫视角）

### 检查项

- **3.1 View Source / 首包 HTML 一致性**  
  - **检查规则（通过）**：对抽样 URL 使用「查看网页源代码」：可见主要文本段落、标题层级、主导航链接；与用户可见主内容无系统性缺失（允许非关键 UI 后hydrate）。  
  - **检查规则（不通过）**：主正文大量依赖客户端后才出现，导致 HTML 中几乎为空或与 title 描述严重不符（易被判定为软薄页或 cloaking 风险）。

- **3.2 元数据在 SSR 阶段输出**  
  - **检查规则（通过）**：`<title>`、meta description、canonical、og/twitter（如使用）在首 HTML 中齐全且与路由匹配。  
  - **检查规则（不通过）**：首 HTML 缺 title/description，仅靠客户端补全。

- **3.3 链接可发现性**  
  - **检查规则（通过）**：重要内链为真实 `<a href>`；关键列表页（分类、百科索引）可被爬虫通过链接遍历到。  
  - **检查规则（不通过）**：大量“假链接”（`div onClick`）承载主导航。

- **3.4 404/410 与软 404**  
  - **检查规则（通过）**：不存在 slug 时返回正确 404；空百科/空列表不应返回 200 且内容为空（易被软 404）。  
  - **检查规则（不通过）**：错误页返回 200 且模板与正常页类似、正文极短。

- **3.5 响应头级指令与 HTML 一致（`X-Robots-Tag` / `Link: canonical`）**  
  - **检查规则（通过）**：若 CDN/Edge/Next `headers()` 下发 `X-Robots-Tag` 或 `Link: rel="canonical"`，则与页面 `<meta name="robots">` / `<link rel="canonical">` **不冲突**；同一 URL 不出现“头里 noindex、正文里 index”之类矛盾。  
  - **检查规则（不通过）**：响应头与 HTML 指令互相否定；或生产环境误带 `noindex`。

- **3.6 移动端与桌面内容对等（移动优先索引下的“可读主内容”）**  
  - **检查规则（通过）**：移动视口下主正文、关键导航、主要内链不被折叠到**爬虫难以稳定获得**的形态（例如默认把全文放进需多次交互才展开且无合理 HTML 承载的容器）；若采用“阅读更多”展开，应确保**主内容仍在 DOM/HTML 中可访问**且非欺骗性隐藏。  
  - **检查规则（不通过）**：移动版主文极大缩水于桌面版（实质性 cloaking/薄移动页）。

### 操作建议（工具）

- `curl -A "Googlebot" -L URL` 对比正常 UA（快速发现明显差异；注意仍要以 GSC 实机渲染为准）。  
- GSC 网址检查中的**网页抓取**与**实际显示**（如有）对照。

## 维度 4：页面体验与 Core Web Vitals（移动端优先）

### 检查项

- **4.1 LCP（最大内容绘制）**  
  - **检查规则（通过）**：博客/百科文章页在 GSC「核心网页指标」中移动端 URL 组不呈现大面积“不佳”；首屏主图/标题区域避免超大未优化图。  
  - **检查规则（不通过）**：首屏 hero 图未限定尺寸/未现代格式/未优先加载策略导致 LCP 长期差。

- **4.2 INP（交互至下次绘制）**  
  - **检查规则（通过）**：目录跳转、折叠 TOC、搜索建议等交互不阻塞主线程到“整页卡顿”。  
  - **检查规则（不通过）**：长任务集中在首屏交互元素上。

- **4.3 CLS（布局偏移）**  
  - **检查规则（通过）**：字体、广告/横幅、嵌入媒体有预留空间；图片带明确宽高或 aspect-ratio。  
  - **检查规则（不通过）**：正文阅读中频繁跳动。

### 操作建议（工具）

- GSC：**体验** / **核心网页指标**；PageSpeed Insights（实验室 + 字段数据）；Lighthouse（移动端）。

## 维度 5：On-page（标题、描述、标题层级、URL）

### 检查项

- **5.1 Title**  
  - **检查规则（通过）**：每页唯一；主意图关键词自然出现且靠前；品牌名统一置后（若采用该模板）；长度避免全站模板化雷同（例如不要 80% 页面都以同一前缀开头）。  
  - **检查规则（不通过）**：title 重复、堆砌、与页面主内容不一致。

- **5.2 Meta description**  
  - **检查规则（通过）**：概括页面独特价值，含 1 次核心措辞即可；避免全站相同描述。  
  - **检查规则（不通过）**：与正文无关或纯关键词列表。

- **5.3 H1 与层级**  
  - **检查规则（通过）**：每页 1 个 H1，与文章/词条主题一致；H2/H3 反映大纲，不跳级混乱。  
  - **检查规则（不通过）**：多个 H1、或 H1 与 title 意图冲突。

- **5.4 URL 可读性与稳定性**  
  - **检查规则（通过）**：slug 英文、短、可读；改名时 301 到新规；避免频繁更换。  
  - **检查规则（不通过）**：无意义 ID slug 且站内大量同类结构导致用户与爬虫都难理解（非绝对不行，但需更强内链与上下文）。

- **5.5 图片与正文媒体（博客/百科高频）**  
  - **检查规则（通过）**：信息性图片有**描述性 `alt`**（一句话说明图中信息与主题关系），非堆砌关键词；纯装饰图使用 `alt=""`。LCP 候选图配合**尺寸/优先级**策略（见维度 4）。示意图/截图尽量有简短说明文字邻近出现。  
  - **检查规则（不通过）**：全站重复 alt、或正文关键信息只存在于图片内而无文字说明；超大未压缩图拖垮 LCP。

- **5.6 Open Graph / Twitter（非排名因子，影响分享预览与部分点击路径）**  
  - **检查规则（通过）**：`og:title`、`og:description`、`og:image`（尺寸合理可拉取）与页面主题一致；避免社交预览与 SERP 严重不一致引发跳出。  
  - **检查规则（不通过）**：缺图导致随机缩略图、或 og 与 canonical URL 指向不一致引发分享链混乱。

## 维度 6：博客/百科内容意图与质量（信息型 SEO）

### 检查项

- **6.1 搜索意图匹配**  
  - **检查规则（通过）**：页面在首屏与前 2 屏内回答该查询的**主问题**；信息层级：定义 → 要点 → 细节 → 延伸阅读。  
  - **检查规则（不通过）**：文不对题、或仅泛泛介绍无实质答案（薄内容）。

- **6.2 覆盖面与可验证性（百科尤甚）**  
  - **检查规则（通过）**：关键主张可追溯到可引用来源；专有名词有清晰定义；避免把推测写为事实。  
  - **检查规则（不通过）**：大量断言无出处；与权威来源明显冲突且无说明。

- **6.3 重复主题站内竞争**  
  - **检查规则（通过）**：多篇文章对应同一主意图时，有明确分工（一篇主文、其他补充角度），并通过内链合并用户路径；必要时合并或 301。  
  - **检查规则（不通过）**：多篇几乎同标题/同大纲互相抢位。

- **6.4 更新与维护策略**  
  - **检查规则（通过）**：对随时间变化的主题显示**更新日期**或“上次审阅”；过时段落修订或加注释。  
  - **检查规则（不通过）**：明显过时信息与当前标准冲突且无提示。

- **6.5 E‑E‑A‑T 信号（经验、专业、权威、可信）**  
  - **检查规则（通过）**：作者/审校者信息（尤其 YMYL 边缘主题）、站点关于页、联系方式/政策页齐全；百科类展示编辑规范更好。  
  - **检查规则（不通过）**：全匿名且主题敏感却无信任背书。

- **6.6 用户优先 vs “写给 Google”（自然度与操纵风险）**  
  - **原则说明**：Google 公开文档与质量评估方向一贯强调**有用、可靠、以用户为中心**的内容；并不存在“偏好某种 SEO 腔调”的单独加分项。技术上做到**可抓取、可理解、不误导**即可；“迎合算法”的堆叠技巧往往带来**质量类算法**或**人工评估**风险。  
  - **检查规则（通过）**：正文读起来像给真实读者解决问题：术语有解释、结构服务理解；关键词出现随话题自然分布；没有为凑词而重复的段落/变体句；可见内容与结构化数据、摘要、标题承诺一致。  
  - **检查规则（不通过）**：明显特征包括但不限于：同义词/长尾变体机械重复；大段与主题弱相关的“SEO 填充段”；列表或 FAQ 仅为覆盖查询而空泛；正文关键词密度异常高于正常写作；隐藏或同色小字；门页式多 URL 指向同一意图；用 AI 生成大量同质页面仅改标题与少量词。  
  - **处置规则**：不通过项优先做**合并/删薄/改写为可验证信息**；技术 SEO（规范 URL、速度、正确 Article 标记）保留，但**不以牺牲可读性与可信度**为代价堆模板化 SEO 段落。

## 维度 7：内部链接与信息架构

### 检查项

- **7.1 重要页面深度**  
  - **检查规则（通过）**：从首页或稳定导航在 ≤3–4 次点击内可达主要分类与代表性文章/词条。  
  - **检查规则（不通过）**：大量孤岛页（无内链入口仅靠 sitemap）。

- **7.2 锚文本多样性**  
  - **检查规则（通过）**：锚文本自然描述目标页主题；避免全站大量相同锚文本硬推同一 URL（易被视作操控信号）。  
  - **检查规则（不通过）**：全站页脚/侧栏重复堆同一关键词锚文本。

- **7.3 相关阅读与主题簇**  
  - **检查规则（通过）**：文章底部/侧栏有“同主题簇”链接，指向更高层 pillar 页或相邻子话题。  
  - **检查规则（不通过）**：相关链接随机或与主题无关（用户体验差，爬虫也难建立主题图）。

- **7.4 分类 / 标签 / 作者归档（薄聚合页治理）**  
  - **检查规则（通过）**：归档索引策略明确：要么提供**独特价值**（可读介绍、精选、定义、内链到代表文章），要么对极低价值归档 **`noindex`** 或合并；分页与 canonical 策略与维度 2 一致。  
  - **检查规则（不通过）**：大量仅罗列标题的归档页被索引且无增量信息，导致站内薄页膨胀与关键词 cannibalization。

## 维度 8：结构化数据（JSON-LD 优先）

### 检查项

- **8.1 Article / BlogPosting（博客）**  
  - **检查规则（通过）**：字段与页面一致：`headline`、`datePublished`、`dateModified`（如有）、`author`、`image`（符合要求）、`mainEntityOfPage`。  
  - **检查规则（不通过）**：时间作者虚假；图片 URL 无效；与可见内容不一致。

- **8.2 百科/定义型页面**  
  - **检查规则（通过）**：仅在页面真实呈现对应内容时，考虑 `DefinedTerm` / `FAQPage` 等；类型选择与页面形态一致。  
  - **检查规则（不通过）**：FAQ 结构化数据承载页面不可见的问答或拼凑问答。

- **8.3 BreadcrumbList**  
  - **检查规则（通过）**：层级与 URL 目录/站点结构一致；面包屑在 UI 上可见或可访问。  
  - **检查规则（不通过）**：面包屑链断、指向错误层级。

- **8.4 校验**  
  - **检查规则（通过）**：Rich Results Test / GSC 增强无严重错误。  
  - **检查规则（不通过）**：大量模板页因同一字段错误批量报错。

- **8.5 Organization / Publisher / Logo（全站一致，可选但推荐）**  
  - **检查规则（通过）**：`Article`/`BlogPosting` 中 `publisher` 指向的 `Organization` 与全站 `logo` URL **稳定可访问**；名称与站点品牌一致。  
  - **检查规则（不通过）**：publisher 缺失或 logo 404；不同模板引用冲突的组织名。

- **8.6 WebSite + `SearchAction`（站内搜索 Sitelinks，可选）**  
  - **检查规则（通过）**：仅当站内搜索**稳定可用**且返回页可索引策略清晰时配置；搜索 URL 不制造无限低质量索引。  
  - **检查规则（不通过）**：搜索产生大量空结果/重复参数页被索引。

## 维度 9：国际化（若仅英文单区域也需核对）

### 检查项

- **9.1 语言声明**  
  - **检查规则（通过）**：`<html lang="en">`（或具体区域 en-US/en-GB）与内容语言一致。  
  - **检查规则（不通过）**：lang 与正文语言不一致。

- **9.2 hreflang（仅多语言/多区域需要）**  
  - **检查规则（通过）**：互指完整、包含 `x-default` 策略清晰；与 canonical 不冲突。  
  - **检查规则（不通过）**：缺互指、或指向不可索引 URL。

## 维度 10：GSC 与 GA4 对齐（发现问题与验收）

### 检查项

- **10.1 着陆页 Join**  
  - **检查规则（通过）**：对同一 URL 规范化（去参）后，GSC 点击高的页面在 GA4 有合理浏览/参与；异常（高点击低参与）能定位到标题党、速度、布局、付费墙等可解释原因。  
  - **检查规则（不通过）**：大量 URL 在 GSC 与 GA4 统计口径严重对不上且无追踪/规范化说明。

- **10.2 查询维度：点击率异常**  
  - **检查规则（通过）**：对高曝光低 CTR 查询，检查 SERP 标题/描述是否不匹配意图；有可迭代实验记录。  
  - **检查规则（不通过）**：仅改标题无内容支撑导致跳出异常升高。

- **10.3 GA4 事件完整性**  
  - **检查规则（通过）**：关键互动（滚动深度分段、站内搜索、订阅/试用等）有事件且参数带上 `page_location`/`content_group` 便于分模板分析。  
  - **检查规则（不通过）**：无法按模板类型拆分博客 vs 百科表现。

## 维度 11：安全、隐私与信任（尤其工具站背景下的内容站）

### 检查项

- **11.1 HTTPS 与混合内容**  
  - **检查规则（通过）**：全站 HTTPS；无 active mixed content。  
  - **检查规则（不通过）**：正文嵌入 HTTP 资源导致浏览器拦截或体验降级。

- **11.2 Cookie / 同意模式对数据的影响**  
  - **检查规则（通过）**：理解 EEA/UK 等地区同意模式导致的 GA4 缺口，不把“数据变少”误判为 SEO 暴跌。  
  - **检查规则（不通过）**：无文档记录导致错误结论。

- **11.3 法律页、政策与可联系性（信任与合规基线）**  
  - **检查规则（通过）**：全站关键入口（页脚/关于）可抵达 **Privacy**、**Terms**（如适用）、**Contact**；邮箱/表单可用；与内容声明（尤其涉及上传/处理数据的工具站背景）无矛盾。  
  - **检查规则（不通过）**：政策页断链、占位符内容、或关键数据处理行为与隐私描述不一致。
